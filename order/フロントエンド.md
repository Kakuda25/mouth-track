このスキルは、ありきたりな「AI っぽい」見た目を避け、独自性がありプロダクション品質のフロントエンドインターフェースを作成するためのガイドです。実際に動作するコードを、審美性と創造性に最大限配慮して実装します。

・デザイン思考

コードを書く前に、以下を理解し、大胆なデザイン方向性を決めます：

目的：この UI は何の問題を解決するか？誰が使うのか？

トーン：極端さを選ぶ。
例：究極のミニマル、混沌のマキシマリズム、レトロフューチャー、オーガニック、ラグジュアリー、遊び心、エディトリアル、ブルータリズム、アールデコ、パステル、インダストリアル……など無限にある。

制約：フレームワーク、パフォーマンス、アクセシビリティなど。

差別化：何が「忘れられない」ものになるのか？ひと目で記憶に残るフックは何か？

重要：はっきりしたコンセプトを選び、それを徹底的に実行すること。
マキシマリズムもミニマリズムも正しい。大事なのは「意図の明確さ」。

・実装の指針

HTML/CSS/JS、React、Vue などで、以下を満たすコードを書きます：

プロダクション品質で機能する

視覚的に強烈で記憶に残る

一貫した美学の方向性

全ディテールまで丁寧に調整されている

フロントエンド美学ガイドライン

重点を置くポイント：

・タイポグラフィ

美しく個性的なフォントを採用

Arial、Inter などの汎用フォントは避ける

ディスプレイ用と本文用を個性的に組み合わせる

・色とテーマ

明確で強いテーマ

CSS カスタムプロパティで一貫性保持

主役カラー＋鋭いアクセント

・モーション

CSS 中心のアニメーション

重要な瞬間（ページロードの段階的登場など）に集中

スクロールトリガー、ホバー演出などで驚きを生む

・空間構成

既成の枠を破るレイアウト

非対称、重なり、斜めの流れ

余白の大胆さ or 密度のコントロール

・背景と装飾

単色ではなく「雰囲気」を作る

グラデーションメッシュ、ノイズ、幾何学パターン、透明レイヤー、粒状感、劇的な影など

カスタムカーソルや質感も検討

※禁止事項（AIっぽさ排除）

Inter / Roboto / Arial など陳腐なフォント

典型的な紫グラデーション＋白背景

平凡なレイアウト・コンポーネント

文脈に寄り添わない画一的なデザイン

※重要

美学の方向性に応じて実装の複雑さも合わせる

マキシマル → コードもリッチに

ミニマル → 控えめだが精密に

意図を完璧に形にすることが鍵。

本気を出せば驚異的な創造力を発揮できる。手を抜かず、徹底して独創的なフロントエンドを作る。
